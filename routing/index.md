# Satoshium Routing Layer

The routing layer connects the Satoshium platform across multiple domains.

Instead of operating as a single monolithic site, Satoshium is structured as a distributed coordination environment. Shared routing infrastructure ensures the ecosystem behaves as a unified platform while remaining modular and extensible.

---

## Purpose of the Routing Layer

The routing layer exists to:

- maintain continuity across domains
- support shared navigation components
- provide consistent platform identity
- coordinate entry points between layers
- reduce duplication across repositories
- enable long-horizon architectural scaling

Routing allows Satoshium to expand without fragmenting.

---

## Shared Platform Components

Several shared components provide structural consistency across the ecosystem.

### Topbar Loader

The topbar loader dynamically inserts the correct navigation surface depending on context:

- platform topbar for `.ai`
- universe topbar for cross-domain surfaces

This keeps navigation aligned without duplicating layout logic in each repository.

---

### Footer Loader

The footer loader ensures consistent:

- mission language
- legal references
- platform links
- structural navigation

across all public-facing domains.

---

### Domain Layers Script

The domain-layers component provides a structured overview of the Satoshium Universe from any page where it appears.

It helps users understand:

- where they are
- how domains relate
- what each layer represents

within the broader architecture.

---

### Sayings Engine

The sayings engine rotates short reflections across platform pages.

These statements reinforce:

- architectural intent
- long-horizon thinking
- platform philosophy
- coordination principles

without repeating static messaging everywhere.

---

## Cross-Domain Coordination

Routing infrastructure supports movement between:

- platform surfaces
- infrastructure layers
- knowledge environments
- experimental labs
- builder interfaces
- registry systems
- Signal Layer coordination pathways

This ensures that domain separation does not produce navigation fragmentation.

---

## Routing and the Signal Layer

Routing also supports visibility into emerging coordination surfaces within the Signal Layer.

As signals move between:

- proposals
- simulations
- governance surfaces
- service definitions

the routing layer helps maintain structured entry points across domains.

This enables coordination before systems become formal platform components.

---

## Why Routing Matters in Phase004

Phase004 establishes Satoshium as a distributed platform rather than a single-site application.

Routing infrastructure makes that possible by:

- preserving architectural clarity
- enabling modular expansion
- supporting shared interface components
- aligning domain responsibilities
- maintaining ecosystem coherence

Routing is not navigation alone.

It is part of the coordination infrastructure of the platform.
