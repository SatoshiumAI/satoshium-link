# Satoshium Link Style Guide

The `satoshium-link` repository documents coordination structures across the Satoshium platform.

Its documentation style emphasizes clarity, structure, and architectural orientation rather than system implementation or conceptual explanation.

This repository helps users understand how the platform fits together.

---

## Purpose of the Coordination Style

Documentation in this repository should:

- explain relationships between domains
- clarify repository roles
- support navigation across platform layers
- maintain structural transparency
- reinforce architectural coherence

It should not duplicate material maintained in system-layer repositories.

---

## Tone and Voice

Use a tone that is:

- precise
- neutral
- structural
- platform-oriented
- forward-compatible

Avoid:

- marketing language
- speculative claims
- implementation-specific instructions
- experimental positioning language

The coordination layer describes structure, not behavior.

---

## Domain References

When referencing platform domains, use consistent naming:

- `satoshium.ai` — platform entry layer
- `satoshium.net` — infrastructure layer
- `satoshium.xyz` — experimental layer
- `satoshium.dev` — builder layer
- `satoshium.info` — knowledge layer
- `satoshium.link` — coordination layer
- `satoshium.store` — cultural layer

Domain references should reflect architectural roles rather than navigation shortcuts.

---

## Repository References

Repository names should be written in lowercase monospace format:

Example:

`satoshium-core`  
`satoshium-trust`  
`satoshium-specs`

Avoid informal abbreviations or alternate naming conventions.

---

## Coordination-Layer Language

Preferred phrases include:

- coordination surface
- routing infrastructure
- domain-layer alignment
- repository mapping
- platform structure
- architectural relationships

Avoid:

- feature descriptions
- implementation walkthroughs
- speculative roadmap language

Those belong in other domains.

---

## Signal Layer References

When describing Signal Layer pathways:

- present them as coordination inputs
- avoid treating them as finalized systems
- emphasize visibility rather than authority
- maintain architectural neutrality

Signal Layer material should remain forward-compatible.

---

## Cross-Domain Consistency

Documentation should align with:

- Architecture pages
- Registry definitions
- Repo Map structure
- Domain-layer documentation
- Routing-layer descriptions

If conflicts appear, architecture and registry surfaces take priority.

---

## Phase004 Alignment

Phase004 establishes Satoshium as a distributed platform composed of coordinated domain layers.

Documentation in `satoshium-link` supports this structure by:

- clarifying relationships
- preserving navigation continuity
- maintaining ecosystem readability
- supporting long-horizon scalability

The coordination layer exists to keep the platform understandable as it grows.
