# Satoshium Link References

The `satoshium-link` repository supports coordination across the Satoshium platform by maintaining routing surfaces, repository mapping structures, and domain-layer documentation.

This document identifies the primary architectural sources that inform coordination-layer alignment.

---

## Purpose of This Reference Document

The coordination layer depends on stable architectural definitions maintained elsewhere in the platform.

These references ensure that routing structures, repo maps, and domain relationships remain consistent with the broader Satoshium ecosystem.

They help maintain:

- platform coherence
- domain-layer clarity
- repository alignment
- cross-surface navigation continuity
- long-horizon architectural traceability

---

## Primary Architecture References

Coordination-layer documentation should remain aligned with:

- the Architecture page
- the Registry
- the Repo Map
- the Domain Layers documentation
- routing-layer definitions
- Signal Layer coordination surfaces

These sources define the structural model that the link domain exposes publicly.

The Architecture surface describes how Satoshium is organized as a layered coordination environment.

- Canon

Coordination documentation should reflect this structure consistently.

- services
- systems
